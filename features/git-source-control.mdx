## Getting started

### Initialize repository

If your folder is not a Git repository:

1. Open the Source Control panel from the sidebar
2. Click **Initialize Repository**
3. DAWNO creates a new Git repository in the current folder

Implementation: `~/workspace/source/renderer/components/SourceControlPanel.tsx:398-403`

### Repository detection

DAWNO automatically detects if the current folder is a Git repository and displays:

- Repository status (changes, staged files, untracked files)
- Commit history
- Incoming/outgoing commits

<Info>
The panel refreshes automatically every 5 seconds to stay in sync with external Git operations.
</Info>

Implementation: `~/workspace/source/renderer/components/SourceControlPanel.tsx:393-396`

## File staging

### Stage individual files

In the **Changes** or **Untracked Files** section:

- Click the **+** button that appears on hover to stage a file
- Click on a file to view its diff
- Right-click for additional options

Implementation: `~/workspace/source/renderer/components/SourceControlPanel.tsx:200-217`

### Unstage files

In the **Staged Changes** section:

- Click the **-** button to unstage a file
- Right-click and select "Unstage Changes"

Implementation: `~/workspace/source/renderer/components/SourceControlPanel.tsx:206-211`

### Bulk operations

Each section has action buttons in the header:

- **Stage All Changes**: Stages all unstaged files (including untracked)
- **Unstage All**: Removes all files from staging area

<Tip>
Use the toolbar "Stage All Changes" button (✓ icon) to stage everything at once.
</Tip>

## Creating commits

### Commit workflow

1. Stage the files you want to commit
2. Type your commit message in the text area
3. Press `Ctrl+Enter` or click the **Commit** button

```
Message (Ctrl+Enter to commit)
```

### Smart staging

DAWNO follows VSCode's behavior:

- If nothing is staged but there are changes, it stages everything before committing
- If files are already staged, only those files are committed

Implementation: `~/workspace/source/renderer/components/SourceControlPanel.tsx:417-462`

<Note>
Commit messages are required - the commit button is disabled when the message is empty.
</Note>

## Context menu actions

Right-click any file for quick actions:

### All files

- **Open Changes**: View diff in modal
- **Open File**: Open in editor
- **Reveal in File Explorer**: Show in system file manager
- **Reveal in Explorer View**: Jump to file in DAWNO's explorer

### Unstaged/Untracked files

- **Stage Changes**: Add to staging area
- **Discard Changes**: Revert to HEAD version (unstaged only)
- **Add to .gitignore**: Ignore this file

### Staged files

- **Unstage Changes**: Remove from staging area
- **Open File (HEAD)**: View the committed version

### Gitignore management

- **Add to .gitignore**: Adds the file path to `.gitignore`
- **Remove from .gitignore**: Removes the entry from `.gitignore`

Implementation: `~/workspace/source/renderer/components/SourceControlPanel.tsx:69-175`

## Stash operations

Save and restore work-in-progress changes:

### Stash changes

1. Click the stash button (📦 icon) in the toolbar
2. All working directory changes are saved
3. Working directory is cleaned

### Pop stash

1. Click the stash pop button (📦↑ icon)
2. Most recent stash is applied
3. Stash is removed from stack

Implementation: `~/workspace/source/renderer/components/SourceControlPanel.tsx:483-495`

<Tip>
Use stash to temporarily save changes when switching branches or pulling updates.
</Tip>
