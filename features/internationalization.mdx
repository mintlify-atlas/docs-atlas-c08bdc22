## Changing language

### In settings

1. Open **Settings** (gear icon or `Ctrl+,`)
2. Scroll to the **Integration** section
3. Find the **Language** dropdown
4. Select your preferred language
5. Click **Save Changes**

The interface updates immediately - no restart required!

<Tip>
Your language preference is saved and persists across sessions.
</Tip>

## Dynamic messages

Many messages use dynamic interpolation:

### Function-based translations

Some translations are functions that accept parameters:

```typescript
window.unsavedChangesConfirm: (count: number) =>
  `${count} file(s) have unsaved changes. Close anyway?`
```

Examples:
- `explorer.deleteConfirm(name)` - "Delete {name}? This will move it to Trash."
- `tabs.unsavedConfirm(name)` - "{name} is unsaved. Close anyway?"
- `configModal.errorLoading(err)` - "Error loading config: {err}"
- `serverModal.failedToStart(err)` - "Failed to start server: {err}"

Implementation: See function signatures in `~/workspace/source/renderer/i18n/locales/en.ts:1-220`

<Info>
Dynamic messages ensure proper grammar and formatting in all languages, even when inserting variable data.
</Info>

## Language architecture

### Type-safe translations

DAWNO uses TypeScript for type-safe translations:

```typescript
export interface Locale {
  menu: {
    file: string;
    build: string;
    settings: string;
  };
  // ... more sections
}
```

This ensures:
- All languages have the same keys
- No missing translations
- IDE autocomplete for translation keys
- Compile-time validation

Implementation: `~/workspace/source/renderer/i18n/locales/en.ts:1-221`

### Language context

Translations are provided via React Context:

```typescript
const { t } = useLanguage();

// Usage:
<button>{t.file.save}</button>
<p>{t.explorer.deleteConfirm(fileName)}</p>
```

### Locale loading

Each language file exports a complete `Locale` object:

```typescript
import en from './locales/en';
import es from './locales/es';
import fr from './locales/fr';
// ... etc
```

The active language is selected based on user settings.

## Adding a new language

To add support for a new language:

1. Create a new file in `renderer/i18n/locales/{code}.ts`
2. Import and implement the `Locale` interface
3. Translate all keys following the English file as reference
4. Export the locale object as default
5. Add the language to the language selector in Settings

<Note>
All language files must implement the complete `Locale` interface - TypeScript will error on missing translations.
</Note>

## RTL support

Arabic (ar) has right-to-left text direction support:

- UI elements flow from right to left
- Text alignment is reversed
- Icons and controls are mirrored appropriately
