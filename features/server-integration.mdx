## Server manager

### Opening the manager

Access the server manager by:

1. Clicking the **Server** button in the toolbar
2. Using the **Build > Server Manager** menu
3. Pressing the server keyboard shortcut

Implementation: `~/workspace/source/renderer/components/modals/ServerModal.tsx:53-112`

### Server detection

DAWNO automatically searches the current project folder for:

- `samp-server.exe` (SA-MP server)
- `omp-server.exe` (open.mp server)

The detected path is displayed in the Server Control section.

<Tip>
If your server executable is in a different location, use the "Change Path" button to manually select it.
</Tip>

### Changing server path

1. Click **Change Path** in the Server Manager
2. Browse for the server executable (`.exe` file)
3. Selected path is saved for the current project

Implementation: `~/workspace/source/renderer/components/modals/ServerModal.tsx:45-51`

## Server control

### Start server

Click the **Start Server** button (▶ icon):

- Server process launches
- Status changes to **ONLINE** (green badge)
- Console output starts streaming

Implementation: `~/workspace/source/renderer/components/modals/ServerModal.tsx:30-34`

<Note>
The Start button is disabled when the server is already running or no server path is detected.
</Note>

### Stop server

Click the **Stop** button (■ icon):

- Server process terminates gracefully
- Status changes to **OFFLINE** (gray badge)
- Console output stops

Implementation: `~/workspace/source/renderer/components/modals/ServerModal.tsx:36-38`

### Restart server

Click the **Restart** button (🔄 icon):

- Server stops
- Server starts again automatically
- Useful for testing script changes

Implementation: `~/workspace/source/renderer/components/modals/ServerModal.tsx:40-43`

<Tip>
Use restart after compiling your script to test changes immediately.
</Tip>

## Error handling

### Failed to start

If the server fails to start:

- Error message appears in the Output panel
- Detailed error reason is shown
- Status remains OFFLINE

Implementation: `~/workspace/source/renderer/components/modals/ServerModal.tsx:33`

### Server not detected

When no server is found:

- Path display shows "Not detected"
- Start and Restart buttons are disabled
- Use "Change Path" to manually select server

Implementation: `~/workspace/source/renderer/components/modals/ServerModal.tsx:64`

<Info>
Make sure your server executable (samp-server.exe or omp-server.exe) is in the root of your project folder for automatic detection.

## Integration with workflow

### Typical development workflow

1. Edit your PAWN script in the Monaco editor
2. Press `F5` to compile the script
3. Check compilation output for errors
4. Open Server Manager
5. Start or restart the server
6. Connect to test your changes
7. View server logs for debugging

### Compile and test cycle

```
Edit → Compile (F5) → Restart Server → Test → Repeat
```

<Tip>
Keep the Server Manager open while developing to quickly restart after each compilation.
</Tip>

## Server process management

### Process lifecycle

DAWNO manages the server process:

- Spawns server as a child process
- Captures stdout and stderr streams
- Monitors process exit events
- Cleans up on application close

### Status change events

The editor listens for server status changes:

- Updates UI badges and buttons
- Enables/disables controls based on state
- Provides visual feedback during state transitions

Implementation: `~/workspace/source/renderer/components/modals/ServerModal.tsx:18-24`

### Log streaming architecture

Server logs are streamed via IPC:

- Main process captures server output
- Sends log entries to renderer process
- Renderer updates UI in real-time
- Auto-scrolls to latest message

Implementation: `~/workspace/source/renderer/components/modals/ServerModal.tsx:14-16`
